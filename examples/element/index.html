Web Component · <lightbox-gallery>

lightbox-gallery

Every feature, one page. The image galleries use the drop-in <lightbox-gallery bind="a"> element; richer cases use the same core it wraps.

Basic Mixed media Slideshow Toolbar Themes Inline Deep-linking API & events

01 Basic gallery

Wrap links in <lightbox-gallery bind="a"> and you have a gallery — no build step required. Click a thumbnail for a FLIP zoom; wheel/pinch zoom, swipe, arrows, f and Esc all work.

Use it for plain HTML pages, CMS content, any framework.

02 Mixed media

One gallery, many types — images, an MP4 video, a YouTube embed and a raw HTML slide, driven by an items array on the core instance.

Use it for mixed image + video galleries, case studies, press kits.

03 Slideshow

Autoplay with a progress bar (slideshow, slideshowDelay, slideshowProgress). Hovering the stage pauses it.

Use it for kiosks, hero showcases, hands-off presentations.

04 Toolbar & actions

The element takes the same options via its options property — fullscreen, download, share, rotate and custom toolbarButtons.

Use it for asset libraries, downloadable galleries, admin tools.

05 Themes

theme: 'dark' | 'light' | 'auto' plus the lbg-theme-glass and lbg-theme-minimal presets. Everything is CSS custom properties.

Use it for matching your site's look, light pages, OS-aware UIs.

06 Inline carousel

inline: true with a container renders the viewer on the page as a zoomable, swipeable carousel instead of a fullscreen overlay.

Use it for product pages, embedded galleries, dashboards.

07 Deep-linking (URL hash)

hash: true writes the open slide to the URL (#gallery=3), so links are shareable and the back button closes the viewer. Open one, then check the address bar.

Use it for shareable photos, SEO-friendly galleries, back-button UX.

08 Programmatic API & live events

Drive the instance from code — open, next, prev, toggleSlideshow, toggleFullscreen — and watch every event fire.

Use it for custom controls, analytics, syncing state elsewhere.
